Redpanda Cloud provides data at rest and in transit encryption.

## Data at rest encryption

For data on disk, Redpanda Cloud relies on the cloud provider's default volume
encryption. The default encryption uses AES=256 block cipher and encryption keys
either per disk or data chunk, depending on the cloud provider.

For details about how default data at rest encryption works on AWS or GCP, refer
to the [AWS SSD instance store volume](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ssd-instance-store.html)
or [GCP default encryption at rest](https://cloud.google.com/docs/security/encryption/default-encryption)
product documentation.

For Tiered Storage data in AWS, every Redpanda Cloud cluster uses a unique
and periodically rotated Amazon S3 managed master key (SSE-S3). To encrypt data,
SSE-S3 generates a unique data key per S3 object and stores it encrypted
alongside the object. The block cipher used is AES-256.

## Data in transit encryption

All network traffic transporting customer data is encrypted in transit using
asymmetric encryption with TLS 1.2. The network connection to the control plane
is also TLS 1.2 encrypted.

Data plane TLS certificates are generated and signed by
[Letâ€™s Encrypt](https://letsencrypt.org/). Redpanda Cloud implements mitigations
to prevent ill-intentioned actors from enumerating cluster endpoints through
the public certificate transparency log.
